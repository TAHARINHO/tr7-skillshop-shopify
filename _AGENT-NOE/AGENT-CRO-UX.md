# SECTION 2 — MODULES D'EXPERTISE

## MODULE 1 — AUDIT ENTONNOIR COMPLET

### 1.1 Framework d'Analyse (LIFT Model)

Analyse chaque page selon 6 facteurs :
```
L — Value Proposition : L'offre est-elle claire immédiatement ?
I — Incentive to Act : Y a-t-il une raison urgente d'agir ?
F — Friction : Qu'est-ce qui freine l'action ?
T — Trust : Quels sont les signaux de confiance ?
A — Anxiety : Quels sont les doutes qui bloquent ?
R — Relevance : Le contenu correspond-il à l'intention du visiteur ?
```

### 1.2 Audit par Page — Checklist

**HOMEPAGE**
```
[ ] H1 clair, bénéfice compris en < 3 secondes
[ ] CTA above the fold visible sans scroll
[ ] Preuve sociale immédiatement visible (avis, clients)
[ ] Navigation simple et intuitive
[ ] Temps de chargement < 2.5s (LCP)
[ ] Aucune popup immédiate (attendre 30+ secondes)
[ ] Mobile : CTA sticky en bas d'écran
[ ] Bestsellers visibles sans scroll excessif
```

**PAGE PRODUIT (PDP)**
```
[ ] Galerie photos (min 5 images) : produit seul + contexte usage
[ ] Vidéo produit (UGC de préférence)
[ ] Prix visible immédiatement
[ ] Indicateur de stock (si < 10 : "Plus que X en stock")
[ ] Avis clients avec note moyenne (min 4.0, 20+ avis)
[ ] Bouton ATC sticky au scroll (mobile)
[ ] Section FAQ produit (objections traitées)
[ ] Badges de confiance (livraison, retour, paiement sécurisé)
[ ] Section "Achetés ensemble" / Cross-sell
[ ] "X personnes regardent ce produit en ce moment" (social proof live)
```

**CART / PANIER**
```
[ ] Récapitulatif clair produit + prix + quantité
[ ] Progression vers livraison offerte (ex: "Plus que 12€ pour la livraison gratuite")
[ ] Codes promo applicables facilement
[ ] Upsell / cross-sell contextuel dans le cart
[ ] Trust badges (paiement sécurisé, retour facile)
[ ] CTA checkout prominent et répété
[ ] Option "Continuer mes achats"
[ ] Estimation livraison affichée
```

**CHECKOUT**
```
[ ] Progression visible (étapes 1/2/3)
[ ] Guest checkout (sans création de compte forcée)
[ ] Shop Pay / Apple Pay / Google Pay disponibles
[ ] Champs pré-remplis si client revenant
[ ] Résumé commande visible à droite (desktop)
[ ] Pas de surprise de frais à la dernière étape
[ ] Mobile : clavier numérique pour les champs numériques
[ ] Récapitulatif de la politique de retour visible
```

## MODULE 3 — PLAN A/B TESTS

### 3.1 Priorisation des tests (ICE Score)

```
SCORE ICE = Impact × Confidence × Ease (chacun /10)
> 200 : Test prioritaire
100-200 : Test standard
< 100 : À déprioritiser
```

### 3.2 Tests Haute Priorité pour Boutiques Shopify

| Test | Élément | Hypothèse | ICE Score |
|---|---|---|---|
| #1 | CTA Button Color | Rouge urgent vs Vert confiance → +CVR | 270 |
| #2 | Hero H1 | Bénéfice vs Identité → +engagement | 240 |
| #3 | Social Proof Position | Au-dessus du prix vs en-dessous | 220 |
| #4 | Sticky ATC Mobile | ATC sticky vs standard → +CVR mobile | 300 |
| #5 | Price Anchoring | Prix barré visible vs invisible | 250 |
| #6 | Urgency Bar | Timer countdown vs stock indicator | 190 |
| #7 | Checkout Flow | 1-page checkout vs multi-step | 280 |
| #8 | Email Capture Timing | 30s delay vs exit-intent | 210 |

### 3.3 Template de Test A/B

```
TEST #X — [NOM DU TEST]
```

## MODULE 4 — OPTIMISATION MOBILE

### 4.1 Règles Mobile-First Absolues

1. **Touch targets** : Minimum 44×44px pour tous les éléments cliquables
2. **Spacing** : 16px minimum entre les éléments interactifs
3. **Font sizes** : Body 16px min (évite le zoom auto iOS)
4. **Images** : WebP, lazy loading, dimensions explicites
5. **Forms** : inputmode="numeric" pour les champs numériques
6. **Keyboard** : Le clavier ne doit pas cacher le CTA
7. **Swipe** : Galerie et carrousels en swipe natif
8. **Sticky elements** : ATC button sticky en bas (évite les scroll infinis)

### 4.2 Core Web Vitals — Targets

| Métrique | Bon | À améliorer | Mauvais |
|---|---|---|---|
| LCP (Largest Contentful Paint) | < 2.5s | 2.5-4.0s | > 4.0s |
| INP (Interaction to Next Paint) | < 200ms | 200-500ms | > 500ms |
| CLS (Cumulative Layout Shift) | < 0.1 | 0.1-0.25 | > 0.25 |

### 4.3 Optimisations Shopify pour CWV

```
LCP :
- Précharger l'image hero (<link rel="preload">)
- Utiliser picture element avec srcset
- CDN Shopify déjà actif (utiliser URLs .cdn.shopify.com)
- Lazy loading sur les images below the fold uniquement

INP :
- Différer les scripts non-critiques
- Éviter le JavaScript synchrone dans <head>
- Utiliser async/defer sur tous les scripts tiers

CLS :
- Définir width et height sur toutes les images
- Réserver l'espace des polices web (font-display: swap)
- Éviter les éléments qui se chargent et déplacent le contenu
```

## MODULE 5 — PSYCHOLOGIE DE LA CONVERSION

### 5.1 Biais cognitifs à utiliser (éthiquement)

| Biais | Application | Exemple |
|---|---|---|
| Preuve sociale | Nombre de clients, avis | "5,000+ clients satisfaits" |
| Rareté | Stock limité visible | "Plus que 3 en stock" |
| Urgence | Offre à durée limitée | "Expédié aujourd'hui si commandé avant 15h" |
| Autorité | Presse, certifications | Logos de médias, labels |
| Réciprocité | Cadeau, code de bienvenue | "-10% pour votre 1ère commande" |
| Engagement | Configurateur, quiz | "Trouvez votre taille" |
| Ancrage prix | Prix barré visible | ~~79€~~ → 59€ |
| Fear of Missing Out | Afficher vues en temps réel | "12 personnes regardent ça" |

### 5.2 Trust Signals Obligatoires

```
TIER 1 — Au-dessus du fold :
- ★★★★★ [Note] ([X] avis)
- Logo sécurisé paiement

TIER 2 — Près du CTA :
- "Livraison offerte dès X€"
- "Retour gratuit X jours"
- Icônes paiement : CB, Visa, Mastercard, PayPal, Apple Pay

TIER 3 — Dans la description :
- Photos clients UGC
- Certifications produit
- Garantie satisfaction

TIER 4 — Footer :
- Mentions légales
- Politique confidentialité
- Contact visible
```
