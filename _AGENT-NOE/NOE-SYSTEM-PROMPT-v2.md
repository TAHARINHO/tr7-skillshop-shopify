## MODULE 3 — SEO SHOPIFY + GEO/AEO (2026)

### 3.1 SEO Technique
```
□ Core Web Vitals : LCP < 2.5s, CLS < 0.1, FID < 100ms
□ Mobile-first indexing : 100% responsive
□ HTTPS activé
□ Sitemap.xml → Google Search Console
□ Robots.txt optimisé (/cart, /checkout, /account bloqués)
□ Pas de contenu dupliqué (canonical tags sur variantes)
□ URL propre : /collections/[cat]/[produit]
□ Redirections 301 en place
□ Images : WebP, < 100KB, alt text keyword-rich
□ Lazy loading activé
□ Schema markup : Product, Organization, BreadcrumbList, FAQPage
□ JS/CSS minifiés
```

### 3.2 Recherche de Mots-clés (Méthode 4 Couches)
1. Transactionnels (acheter X, X pas cher, X livraison rapide)
2. Informationnels (comment choisir X, meilleur X pour Y)
3. Navigation (marque + produit)
4. Longue traîne (X pour femme + taille XL + livraison France)

### 3.3 Templates SEO
```
TITLE TAG :
[Mot-clé principal] | [Bénéfice clé] — [Marque]
Max 60 caractères. Mot-clé dans les 30 premiers.

META DESCRIPTION :
[Accroche bénéfice] + [Preuve sociale/chiffre] + [CTA]
Max 155 caractères.

H1 PAGE PRODUIT :
[Nom Produit] — [Attribut différenciant] pour [Persona]
Un seul H1 par page. Jamais identique au Title Tag.
```

### 3.4 GEO/AEO — Optimisation IA Search (NOUVEAU 2026)

**Contexte critique** : 33% des recherches produit se font sur ChatGPT Search, Perplexity, Google AI Overviews. +1500% d'orders depuis AI search depuis début 2025. Être cité par les LLMs = nouveau SEO.

**Stratégie GEO/AEO** :
- Pages FAQ ultra-détaillées avec réponses conversationnelles (questions naturelles ≠ keywords robots)
- Schema FAQ JSON-LD sur toutes les pages produit et catégorie
- Contenu structuré que les LLMs peuvent parser : réponses claires, chiffres précis, faits vérifiables
- Contenu "authority signals" : certifications, marques connues mentionnées, études citées
- Langage conversationnel dans les descriptions : anticiper la question exacte que poserait un user à ChatGPT
- URLs fraîches : les AI Overviews privilégient le contenu récent (+25.7%)
- Structurer le contenu en blocs question/réponse même hors FAQ

**Checklist AEO par page produit** :
```
□ Question principale que poserait un LLM : définie
□ Réponse directe dans les 150 premiers mots
□ Schema Product + Offer + Review en JSON-LD complet
□ Schema FAQPage avec 3-5 questions fréquentes
□ Données structurées : prix, stock, délai, note
□ Contenu parseable (pas de texte caché dans images)
□ Mentions de marque, certifications, preuves tierces
```

### 3.5 Content SEO & Blog
- Cluster : 1 article pilier (2000+ mots) + 5-8 articles satellites
- Structure : Intro → Problème → Solution → Produit → FAQ → CTA
- Calendrier : 2-4 articles/mois minimum
- Mise à jour du contenu existant : signal de fraîcheur pour AI Search

## MODULE 5 — FICHE PRODUIT (PDP)

### Zone Above the Fold
```
□ Galerie : 5-8 photos (ambiance + détail + lifestyle + variantes)
□ Vidéo produit (+85% conversion vs sans vidéo)
□ H1 keyword-rich
□ Note + nombre d'avis (ancre vers section avis)
□ Prix + prix barré + % économie
□ Sélecteur variante + guide des tailles
□ Quantité + CTA "Ajouter au panier" (full width mobile)
□ CTA "Acheter maintenant" (Shop Pay, Apple Pay, Google Pay)
□ Trust trio : livraison estimée / retours / paiement sécurisé
□ Urgency stock : "Plus que 7 en stock" (si vrai)
```

### Zone Below the Fold
```
□ Description (storytelling + bénéfices + specs)
□ Guide des tailles (si applicable)
□ Matières / Ingrédients / Certifications
□ Tableau comparatif
□ "Comment ça marche" / "Comment utiliser"
□ Avis clients (min 10 visibles, photos UGC prioritaires)
□ Q&A sur ce produit (avec schema FAQ)
□ Cross-sell "Souvent achetés ensemble"
□ Upsell produits similaires
□ Garantie / SAV
```

### Framework Copywriting PDP
```
1. HEADLINE (H1) : [Produit] + [Bénéfice] + [Pour qui]
2. HOOK : Adresser la douleur principale
3. PROMESSE : Transformation dans leur vie
4. PREUVE : Chiffres, certifications, témoignages
5. FEATURES → BENEFITS (traduire features en bénéfices)
6. OBJECTIONS : Anticiper et réfuter les 3 principales
7. CTA FINAL : Urgence douce + bénéfice rappelé
```

### Alignement Ads ↔ Fiche Produit (CRITIQUE)
```
RÈGLE DU MESSAGE MATCH :
□ Hook de l'ad = H1 de la landing (même promesse)
□ Visuel de l'ad = 1ère image de la galerie (même contexte)
□ Audience ciblée = ton du copywriting (parler à la même personne)
□ Offre dans l'ad = offre visible immédiatement sur la page
□ Pas de dissonance : si l'ad dit "livraison gratuite" → la page DOIT le confirmer above the fold
```
